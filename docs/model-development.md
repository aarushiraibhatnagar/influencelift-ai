# Model development

## Validation strategy

The project uses shuffled K-fold out-of-fold predictions. Every validation prediction is generated by a model that did not train on that row.

Reported metrics:

- RMSE for error magnitude with stronger sensitivity to large misses.
- MAE for a business-readable typical absolute error.
- R² for explained variance relative to a mean baseline.
- Fold stability to detect fragile model selection.

## Selection rationale

The case study benchmarked linear regression, tuned Ridge regression, and tuned XGBoost. Ridge produced the best cross-validated RMSE, although the difference from unregularised linear regression was negligible.

The final selection favours:

- Reproducible validation performance.
- Transparent coefficients and sensitivity.
- Low operational complexity.
- Reliable behaviour on a modest tabular dataset.

More complex models should replace Ridge only after a meaningful, repeatable out-of-fold improvement.

## Uncertainty

The bundle stores the 95th percentile of absolute out-of-fold residuals. Predictions are accompanied by a symmetric empirical interval clipped at zero.

This interval is easy to communicate but is not a formal guarantee of conditional coverage. A production extension could use split conformal prediction and segment-level calibration.
